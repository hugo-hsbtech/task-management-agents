<objective>
Wire the Work Item Orchestrator (Plan 02) into the operator-facing CLI: three new Typer subcommands (`hsb run-next-step`, `hsb show-state`, `hsb show-next-action`) and a repo-root `run_loop.py` thin wrapper. <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator CLI input → Typer handler args | `--work-item-id` is a string from the shell; mitigated by Pydantic validation downstream when constructing input contracts |
| run_loop.py subprocess output → loop driver | `subprocess.run` `returncode` is the only signal trusted; non-zero terminates loop (Pitfall 5) |
| Linear MCP read response → show-state Table render | Linear data is rendered into terminal output; no eval, no shell escape — `console.print` is safe |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3-02 | Information Disclosure | CLI command output (run-next-step, show-state) | mitigate | All `print` and `console.print` calls render only work_item_id, status fields, PR URLs. No `ANTHROPIC_API_KEY`, `LINEAR_API_KEY`, full prompt content, or filesystem paths leak through stdout. |
| T-3-03 | Denial of Service | run_loop.py infinite loop on error | mitigate | `result.returncode != 0` check after every `subprocess.run` call (Pitfall 5). On non-zero exit, print error and `sys.exit(returncode)` — no retry, no continuation. KeyboardInterrupt also exits cleanly. |
| T-3-04 | Elevation of Privilege | show-next-action read-only contract | mitigate | `_render_next_action` MUST NOT call any Linear write operation. Test `test_show_next_action_no_side_effects` patches `run_validated_linear_agent` and asserts no `update`/`create`/`comment` operation appears in any call. CLIR-03 contract verified at unit test boundary. |
</threat_model>
