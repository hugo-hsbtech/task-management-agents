1"""Runtime-agnostic Protocol and option shape. 

2 

3Historically the lowest common denominator between ``claude_agent_sdk`` and 

4``openai_codex_sdk``. The canonical home for vendor-neutral types is now 

5the :mod:`llm_providers` library — this module re-exports those types 

6where the field shapes line up, and keeps the legacy ``AgentOptions`` 

7dataclass intact for callers (and tests) that still construct it with the 

8older field set. 

9 

10Deprecation window: 

11 

12* :data:`Message` is the library ``Message`` (identical fields — ``text``, 

13 ``is_final``, ``raw``). 

14* :data:`PermissionMode` is the library ``PermissionMode`` 

15 (``Literal["default", "acceptEdits", "plan", "bypassPermissions"]``). 

16* :data:`Runtime` is aliased to :class:`llm_providers.base.BaseProvider` 

17 so ``isinstance(x, Runtime)`` works for new provider instances. The 

18 legacy compat shims (``ClaudeRuntime`` / ``CodexRuntime``) are 

19 structural duck-types and intentionally do not subclass 

20 ``BaseProvider``. 

21* :class:`AgentOptions` keeps the legacy field set 

22 (``system_prompt: str``, ``allowed_tools: tuple[str, ...]``, 

23 ``mcp_servers: dict | None``, ``hooks: Any``) so the compat shims and 

24 ``make_agent_options`` factory continue to work. Migration to 

25 ``ProviderOptions`` lands as part of Task 21+. 

26"""

49 """Legacy runtime-agnostic option shape returned by ``make_agent_options()``. 

50 

51 Kept as a distinct dataclass during the deprecation window because the 

52 field set differs from :class:`llm_providers.protocol.ProviderOptions` 

53 (string ``system_prompt`` vs ``SystemPrompt`` sum type, ``dict``

56 directly; the compat shims in :mod:`hsb.runtime.compat` translate 

57 instances of this class into each SDK's native shape.
